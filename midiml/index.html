MIDIML

An xml-like markup language that makes it easier to define and build multi-track MIDI files

More on Github

Try it yourself

program.midiml 
<MIDIProgram>
            <Track tempo="100">
                <Notes duration="1" interval="1" vol="100" >
                50, 52, 54, 55, 57, 59, 61, 62, 64, 66, 64, 62, 61, 62, 61, 59
                </Notes>
            </Track>
            <Track tempo="100">
                <Notes duration="2" interval="4" vol="80">
                [50, 54, 57], [50, 54, 57], [52, 55, 59], [52, 55, 59], [50, 54, 61]
                </Notes>
            </Track>
        </MIDIProgram>

python3 midiml.py

Documentation

  • Program must be wrapped in a <MIDIProgram> tag for the interpreter to recognize it
  • <Track> tag defines a MIDI track. You can specify the tempo of the track here.
  • <Notes> tag defines a note sequence. You can specify the duration of the note, the interval of time in between each note, and the volume here.
  • Note sequences can be comma separated integer lists (single note sequences) or comma separated integer arrays (chord sequences)
The program needs to be defined in a filename ending with .midiml, and you need to aim at the midiml file in midiml.py. When midiml.py is ran, a .mid file will be outputted, which can then be played with any midi synthesizer of your choice.
